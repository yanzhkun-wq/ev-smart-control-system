# 硬件能力与客户功能清单（用户已提供）

> 来源：厂家宣传/规格说明（产品特点、规格表、定位功能表）。  
> **约定**：下列能力在后端 + 未来 App 中均需实现；808 细节以 `docs/protocol/` 内厂家资料为准，部标 JT/T 808 为基线。

## 一、产品特点（硬件侧）

| 类别 | 内容 |
|------|------|
| 联网与设计 | 4G、内置天线、高集成设计 |
| 电源 | 9–90V 宽电压；内置备用电池（规格表：55mAh / 3.7V） |
| 定位 | 实时定位追踪；轨迹回放 |
| 告警 | 震动报警；超速报警；进出电子围栏报警 |
| 安全 | 非法拆除立即告警 |
| 定位性能 | 搜星快；Wi‑Fi + GPS + 北斗 + LBS 多重定位 |

## 三、需在平台实现的功能（规格表「功能」）

- **电源检测**：主电断开时设备上传报警，平台接收、展示、推送。
- **数据上报**：GPS 等数据实时上报；支持**客户可配置**的上报策略（间隔、事件触发等，以协议与厂家能力为界）。
- **电子围栏**：规格表标注 TBD — 平台需支持围栏配置、下发（若协议支持）、进出场告警处理与查询。
- **驾驶行为**：车速超过设定阈值 → **超速报警** 上报与通知。
- **GPS 数据**：持续实时位置与状态。
- **震动报警**：设防状态下震动触发告警；平台需支持**设防/撤防**状态管理与告警链路。

## 四、定位功能表（七项）

1. **定时跟踪**：按预设间隔回传经纬度等定位信息；平台可配置间隔（在协议允许范围内）。
2. **街景地图**：App/地图侧展示（对接地图 SDK；街景能力依赖地图服务商）。
3. **超速报警**：超限速告警到手机/平台。
4. **震动报警**：持续振动立即告警提示。
5. **电子围栏**：驶出预设区域 → 告警到管理平台（及 App）。
6. **历史轨迹**：至少支持 **365 天**级别轨迹存储与回放（速度、方向、停顿时长等）。
7. **车队管理**：一机多车 / 一车多账号等灵活绑定与权限（业务与数据模型需预留）。

## 五、与 JT/T 808 的对应关系（实现时核对厂家文档）

以下仅为常见部标对应，**消息号与附加信息 ID 以厂家协议为准**：

| 业务能力 | 808 侧常见切入点 |
|----------|-------------------|
| 实时位置、多重定位、定时跟踪 | 位置信息汇报（如 **0x0200**）；解析状态位、报警标志、附加信息项（卫星数、基站、Wi‑Fi 等若上报） |
| 超速、震动、围栏、主电拆除、非法拆卸等 | **报警标志/事件**；附加信息中的报警描述；部分告警为独立事件消息（查厂家表） |
| 围栏下发、终端参数 | 平台下发终端参数、区域设置等（部标中有围栏/圆形区域等指令集，以选用版本为准） |
| 设防/撤防 | 可能为终端参数、控制指令或私有扩展 — **必须以厂家文档为准** |
