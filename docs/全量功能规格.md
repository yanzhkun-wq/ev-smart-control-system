# 电动车智控系统 — 全量功能规格

> **依据**：（1）三张产品/规格/定位功能说明图；（2）`docs/protocol/WZ标准版808协议V28.pdf`（WZ 通用版部标 JT/T808 **V2.8**）。  
> **用途**：本文件为「要开发的系统」的完整功能清单；协议位定义与报文细节以 PDF 及 `WZ-808-V28-summary.md` 为准。  
> **范围**：含 **设备接入网关、业务后端、管理端、未来 App**；电子围栏在硬件规格表中为 TBD，平台仍按下列「围栏与区域」能力设计，终端侧是否支持下发以厂家后续文档/实测为准。

---

## 一、系统总体目标

| 编号 | 能力 |
|------|------|
| G-01 | 平台作为 **TCP 服务端**，终端按 JT/T808（WZ V2.8）长连接接入。 |
| G-02 | 业务与存储与具体厂家解耦：**协议解析在网关/适配层**，上层为统一领域模型（终端、车辆、位置点、告警、指令、用户组织）。 |
| G-03 | 支持 **实时监管、历史追溯、告警处置、远程控制与参数配置、车队与多用户**。 |
| G-04 | 对外提供 **HTTP/WebSocket 等 API**，供 **Web 管理端与未来 App** 使用（终端永不直连 App）。 |
| G-05 | 支撑产品宣传能力：**4G、宽电压设备、多重定位、震动/超速/围栏/主电与拆除等告警、设防撤防、轨迹回放（目标 365 天）、车队管理**。 |

## 二、设备接入与协议栈（网关层）

| 编号 | 功能 | 协议/说明 |
|------|------|-----------|
| P-01 | TCP 接入、会话管理（单终端连接策略、超时、断线重连兼容）。 | JT/T794 相关约定 |
| P-02 | 帧边界：`0x7e`、转义还原、`0x7d 0x02/0x01` 规则。 | PDF 1.4 |
| P-03 | 消息头解析：消息 ID、消息体属性（含分包 bit13）、终端手机号 BCD、流水号、分包封装项。 | PDF 表 2–3 |
| P-04 | 校验码：从消息头第一字节至校验码前一字节异或。 | PDF 1.4.4 |
| P-05 | 分包报文重组与顺序处理。 | 属性字 bit13 |
| P-06 | 消息路由：按 ID 分发至解码器；厂家扩展预留钩子。 | WZ V2.8 |
| P-07 | 向终端发送平台消息：组帧、转义、写流水号、可选加密预留。 | 与 P-02 对称 |
| P-08 | **客户兼容配置**可落库：不同终端可能使用不同报警位/状态位/扩展字段策略（PDF 第 3 章，如 `808_SFFLAG`、`VIB_ALARM_808`、`808_LBS`、`WIFI_ENABLE`、`808_SEL`、`NO_UP_ICCID`、`808_FIXED_IP`、`EXIO_FUN` 等）。平台需能按终端或客户维度记录并在解析时应用（或委托设备出厂固定配置 + 平台只读展示）。 | PDF §3 |

**需在网关/适配中实现的消息（与 PDF 一致，含扩展能力）**  

`0x0001` 终端通用应答 · `0x8001` 平台通用应答 · `0x0002` 心跳 · `0x0100` 注册 · `0x8100` 注册应答 · `0x0102` 鉴权 · `0x8103` 设置终端参数 · `0x8104` 查询终端参数 · `0x0104` 查询终端参数应答 · `0x8105` 终端控制 · `0x0200` 位置汇报 · `0x8201` 位置查询 · `0x0201` 位置查询应答 · `0x0704` 定位批量/盲区补传 · `0x8300` 文本下发 · `0x6006` 文本上报 · `0x0900` 上行透传 · `0x8900` 下行透传  

（若后续厂家补充圆形/矩形区域等指令，在「围栏与区域」中增列并迭代网关。）

## 三、终端生命周期与会话

| 编号 | 功能 | 协议 |
|------|------|------|
| L-01 | 处理 **终端注册**（省市区、制造商、型号、终端 ID、车牌等），登记或更新终端档案。 | 0x0100 |
| L-02 | 返回 **注册应答**：结果码、鉴权码（STRING）；失败分支可运营介入。 | 0x8100 |
| L-03 | 处理 **鉴权**（复位后重连、鉴权码校验）。 | 0x0102 |
| L-04 | **心跳**维持会话；平台 **通用应答**。 | 0x0002 / 0x8001 |
| L-05 | **通用应答**与流水号匹配：终端对平台、平台对终端。 | 0x0001 / 0x8001 |
| L-06 | 业务侧 **在线/离线** 判定、最近一次通信时间、可选 IP 记录。 | — |
| L-07 | 终端复位后重新注册流程与数据一致性（PDF：复位后都会重新注册）。 | 0x0100 |

## 四、定位、跟踪与轨迹（对齐产品与协议）

### 4.1 实时定位

| 编号 | 功能 | 协议/数据 |
|------|------|-----------|
| T-01 | 解析 **0x0200 位置基本信息**：报警标志、状态、经纬度（×1e6）、高程、速度（1/10 km/h）、方向、时间（BCD GMT+8）。 | 0x0200 表 16 |
| T-02 | 解析 **状态位**：ACC、定位有效、经纬方向、**设防/撤防**、油路、**主电断开**、车门、**GPS/北斗** 等。 | 表 17 |
| T-03 | 解析 **报警标志位**：紧急、**超速**、疲劳、主电欠压、**主电源断开**、无线低电、**震动**、**拆除**、超时停车、非法位移、车门开、**ACC 开/关**、设备开/关机等；**V2.8 无线设备开关机相关位**以 PDF 最新表为准。 | 表 18 |
| T-04 | 解析 **位置附加信息 TLV** 全量（至少支持 WZ 表 20 已列 ID）：里程 0x01、CSQ 0x30、卫星数 0x31、温度 0x51、2G 基站 0x53、Wi‑Fi 0x54、电量 0x56（含 1–100% 字节）、4G 基站 0x5D、主电电压 0x61、ICCID 0xF1、设防扩展 0xF3、**扩展报警 0xF4**、gSensor 0xF5、无线模式/定位/充电 0xF6、网络类型 0xF7、休眠 0xF8、ACC 累计 0xF9、RFID 0xFA 等。 | 表 20 |
| T-05 | **多重定位展示**：地图/接口层可呈现 GPS/北斗/Wi‑Fi/基站/LBS 辅助信息（与产品「Wi‑Fi+GPS+北斗+LBS」一致）。 | 0x31/0x53/0x54/0x5D/0xF6/0xF7 |
| T-06 | **位置查询**：平台主动查位、终端应答同 0x0200 结构。 | 0x8201 / 0x0201 |
| T-07 | **批量/补传**：处理 0x0704 多条位置项（含盲区补传类型）。 | 0x0704 表 26–27 |

### 4.2 上报策略（可配置 / 客户定制）

| 编号 | 功能 | 协议 |
|------|------|------|
| T-08 | 支持通过终端参数配置：**定时/定距/组合汇报**、休眠间隔、缺省时间/距离间隔、拐点补传角度等。 | 0x8103 参数 0x0020/0x0027/0x0029/0x002C/0x0030 等 |
| T-09 | 支持 **查询终端参数** 与应答展示/对比。 | 0x8104 / 0x0104 |
| T-10 | 与产品「**客户定制数据上报**」对齐：在协议允许范围内提供运营/客户可配模板或策略（具体交互在管理端/App 实现）。 | 0x8103 等 |

### 4.3 历史轨迹与回放

| 编号 | 功能 |
|------|------|
| T-11 | 持续写入轨迹点（时间、位置、速度、方向、状态/报警快照、关键附加信息）。 |
| T-12 | **历史轨迹查询与回放**（产品目标：**不少于 365 天**量级，需存储与归档策略）。 |
| T-13 | 回放能力：**速度、方向、停顿时长**等（对齐「历史轨迹」功能表）。 |
| T-14 | **定时跟踪**：与终端上报间隔、平台展示刷新一致（产品功能表第 1 条）。 |

### 4.4 地图与街景

| 编号 | 功能 |
|------|------|
| T-15 | 实时地图：多车/多终端、图标状态（在线、告警、设防等）。 |
| T-16 | **街景地图**入口或坐标跳转（依赖第三方地图/街景 SDK；非 808 内建）。 |

## 五、告警、事件与通知

| 编号 | 功能 |
|------|------|
| A-01 | 告警统一模型：来源（报警 DWORD / 0xF4 / 状态位）、级别、开始时间、终端、是否需确认。 |
| A-02 | 告警列表、筛选、详情（关联当时轨迹点/快照）。 |
| A-03 | **告警推送**：App / 短信 / 微信等（具体通道在集成阶段选型）。 |
| A-04 | 与部标一致的 **报警处理确认**（平台通用应答结果码等，按 PDF 2.2）。 |
| A-05 | **超速报警**：与参数最高速度 0x0055、持续时间 0x0056 及报警位联动展示与统计。 |
| A-06 | **震动报警**：与 **设防** 状态（状态位 bit6、0xF3）联动。 |
| A-07 | **主电断开/欠压**、**非法拆除**、**车门开**、**非法位移** 等安全类告警。 |
| A-08 | **ACC 开/关报警**、**设备开/关机**（含 V2.8 无线开关机相关）。 |
| A-09 | **扩展报警 0xF4**：急加减速、急刹、急转弯、碰撞、侧翻、IO、高温、举升等。 |
| A-10 | **主电断开上报方式**兼容（持续 vs 仅 3 次，`POWER_ALARM_808`）。 | PDF §3.2 |
