## 3. 项目简介（What does your project do?）

```
EV Smart Control System 是一个开源的电动车智能安全防护平台。它通过 GPS + 
北斗多重定位、电子围栏、震动/拆卸传感器等技术，在电动车被盗、电池被拆 
等异常情况下，第一时间向车主和管理平台发送实时告警。

项目基于国标 JT/T808 协议，兼容市面上主流定位硬件设备，支持远程锁车、
轨迹追踪、车队管理等完整功能。用户可以用自己的服务器搭建，数据完全自主
掌控，无需依赖任何商业第三方平台。

目标用户：电动车车主、社区安保、物业公司、校园/园区车队管理者。
```

---

## 4. 项目的社会意义（Why is this project important?）

```
在中国，电动自行车保有量超过 3.5 亿辆。每年被盗案件数以百万计，电池被盗
更是高频事件。然而市面上的防盗定位方案几乎全是商业闭源产品——设备贵、
月费高、数据存储在厂商服务器，用户隐私无法保障。

这个项目旨在打破这一困境：
1. 安全普惠 —— 零软件成本，任何有技术能力的社区或个人都能部署
2. 隐私自主 —— 数据归用户所有，不经第三方
3. 开放兼容 —— 基于国标 JT/T808，不绑定特定硬件品牌
4. 社区共建 —— 开源协作，持续迭代，任何人都能贡献和改进

我们相信：安全不应该只属于买得起昂贵服务的人。开源让安全普惠成为可能。
```

## 7. 计划如何使用 Codex（How will you use Codex?）

```
Codex 将在以下方面加速项目开发：

1. 协议完善（JT/T808）
   - 国标 JT/T808 协议有上百种消息类型，目前仅实现了核心部分
   - 利用 Codex 快速补全剩余消息类型的编解码逻辑（注册、鉴权、
     位置上报、报警、控制指令、参数配置等）

2. 数据持久化层
   - 设计并实现 Redis + PostgreSQL 存储方案
   - 365 天历史轨迹数据的存储、索引、查询优化

3. 多厂家适配
   - 不同硬件厂商的协议扩展各不相同
   - 用 Codex 加速编写适配器，降低新设备接入门槛

4. 前端功能完善
   - 管理后台的实时地图、轨迹回放、围栏绘制等功能
   - 微信小程序的告警推送、设备绑定等交互体验优化

5. 代码审查与重构
   - 利用 Codex 进行代码审查，提升代码质量
   - 重构早期原型代码，建立更好的架构基础

作为独立开源维护者，Codex 将显著提升开发效率，让我能更快地为社区
交付一个真正可用的产品。
```
