# 参与贡献指南

感谢你对 EV Smart Control System 的关注！我们欢迎各种形式的贡献。

## 🌟 如何贡献

### 报告 Bug
1. 在 [Issues](../../issues) 中创建新 Issue
2. 描述问题现象、复现步骤、期望行为
3. 附上相关日志或截图

### 提出新功能
1. 先搜索 [Issues](../../issues)，确认没有重复
2. 创建 Feature Request，说明使用场景和价值

### 提交代码
1. **Fork** 本仓库
2. 创建功能分支：`git checkout -b feat/your-feature`
3. 遵循现有代码风格
4. 提交前确保代码能正常运行
5. 提交 PR，描述改动内容和原因

## 📁 项目架构

```
用户层
├── 微信小程序 (uni-app)    ← 车主使用
├── Web 管理后台 (React)     ← 管理员使用
└── 未来 App                ← 规划中

网关层
└── Device Gateway (Node.js) ← JT/T808 TCP 服务器
    ├── 帧解析器 (0x7e 转义还原)
    ├── 消息路由 (按 ID 分发)
    └── 厂家适配钩子

数据层
└── 待实现（Redis + PostgreSQL 方案规划中）
```

## 🛠 开发环境

- **Node.js** 18+
- **微信开发者工具**（小程序调试）
- **Docker**（可选，用于部署测试）

## 📝 代码规范

- TypeScript 优先，JavaScript 也可
- 使用项目现有的 ESLint/Prettier 配置（如有）
- 注释清晰，关键逻辑必须注释

## 🎯 优先贡献方向

| 优先级 | 方向 | 说明 |
|--------|------|------|
| 🔴 高 | 协议完善 | JT/T808 更多消息类型支持 |
| 🔴 高 | 数据持久化 | 数据库方案设计与实现 |
| 🟡 中 | 前端优化 | 管理后台和移动端体验提升 |
| 🟡 中 | 厂家适配 | 更多设备型号接入 |
| 🟢 低 | 文档翻译 | 英文等多语言支持 |

## 📞 沟通

- 优先通过 GitHub Issues 沟通
- 重大变更请先开 Issue 讨论再动手
